# Banking Risk Analysis Project (MySQL)

## 📌 Project Overview

This project focuses on analyzing banking customer data using MySQL.

The main objective was to clean raw banking data, handle missing values, validate business logic, and perform analysis using SQL queries.

The project includes:
- Data Cleaning
- Data Standardization
- Business Validation
- Exploratory Data Analysis (EDA)

---

# 📁 Dataset Overview

The dataset contains banking customer and transaction details such as:

- Account_ID
- Customer_Name
- Age
- Gender
- Account_Type
- Balance
- Transaction_Amount
- Transaction_Type
- Loan_Status
- Credit_Score
- KYC_Status
- Account_Status
- Branch

# 🔧 Project Workflow

## 1️⃣ Database Setup
- Created database
- Imported dataset
- Created staging table for cleaning

## 2️⃣ Data Cleaning
- Checked duplicates
- Removed extra spaces using TRIM()
- Converted blanks into NULL values
- Handled missing values
- Standardized gender values

## 3️⃣ Business Validation
- Validated Debit/Credit transaction logic
- Checked KYC and Loan_Status inconsistencies
- Identified negative balances

## 4️⃣ Exploratory Data Analysis (EDA)

### 📊 Problem Statements Solved

- Identify high-risk customers
- Detect fraudulent transactions
- Analyze customer segmentation
- Improve loan approval decisions
- Monitor account inactivity
- Evaluate branch performance
